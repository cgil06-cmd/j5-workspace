# THE REFINERY
## J5 Workflow for Turning Inspiration Into Formation and Execution
## Built on the Deep-Dive Analysis Framework | Created: 2026-03-01

---

## What This Is

Curtis consumes a lot. Books, frameworks, podcasts, programs, people, YouTube transcripts, zip files.
Without a system, inspiration piles up and fades. With this system, every piece of content
Curtis touches becomes part of a growing, connected body of wisdom — tagged, searchable,
mapped to his theology, his business, and his life.

The Refinery doesn't just store content. It transforms it.
Raw inspiration → filtered insight → Curtis's own synthesis → actionable output.

Trigger: Curtis sends any of the following:

## The Six Phases

### Phase 1 — INGEST & SECURE
*Never lose it. File it immediately.*

1. Save all files to `brain/inspiration/[source-name]/` with today's date
2. Extract and read all text content (markdown, docx, PDFs, transcripts)
3. If URL: fetch full content, save locally
4. If YouTube: pull transcript, save as markdown
5. If voice note: transcribe, save as markdown
6. Confirm to Curtis: "Stored. Analyzing."

### Phase 2 — FOUNDATIONAL RESEARCH
*Cast a wide net. Understand the full landscape before analyzing.*
*(From the Deep-Dive Analysis Framework)*

1. **Who is this person/program?** — Background, audience, core thesis, business model
2. **What are they actually saying?** — Core frameworks, key concepts, signature language
3. **What's their evidence base?** — Anecdote? Research? Personal experience? Client results?
4. **Who else is saying this?** — Where does this fit in the broader conversation?
5. **What's their blind spot?** — What are they NOT addressing? What do they assume?
6. **Search for critiques** — What do smart people push back on?

Save all research to `brain/inspiration/[source]/research/` files.

### Phase 3 — SYNTHESIS & FRAMEWORK MAPPING
*Move from raw information to structured knowledge.*
*(Extended from the Deep-Dive Analysis Framework)*

Create `brain/inspiration/[source]/synthesis.md` with these sections:

**3a. Core Framework Summary**
- What is the central idea in one sentence?
- What problem does it solve?
- What's the transformation it promises?

**3b. Map Against Curtis's Existing Frameworks**
Always check against:
- SOUL.md — does this align with or challenge Curtis's identity and values?
- F&F Theology (Simul Justus et Peccator, The F&F Loop, The Four Faces)
- His operating calendar and rhythms
- J5 system architecture
- His discipleship framework for Caden and Chase

**3c. The Curtis Lens**
Ask these questions explicitly:
1. What does Curtis already know that this confirms?
2. What does this challenge or complicate?
3. Where does Curtis's theology go *further* or *deeper* than this framework?
4. What would Curtis say to a pastor who brought him this framework?
5. Is this behavioral (what you do) or formational (who you're becoming)?

**3d. Steal-Worthy Elements**
List only the highest-leverage ideas. Max 5. Be specific:
- What exact concept, tool, or language is worth adopting?
- How would Curtis use it — in F&F, in his life, in J5?

### Phase 4 — THE CURTIS SYNTHESIS
*This is the money. Not "here's what they said." Here's what Curtis thinks.*

Write a 500-800 word synthesis document that:
- Opens with Curtis's take, not the source's
- Shows where this framework fits in Curtis's larger worldview
- Names where Curtis agrees, disagrees, and extends
- Identifies the F&F angle (what product/content/framework could emerge from this?)
- Identifies the personal formation angle (what is Curtis being invited to?)
- Identifies the J5/operational angle (what should be built or changed?)

This document becomes a permanent part of Curtis's intellectual library.
File at: `brain/inspiration/[source]/curtis_synthesis.md`

### Phase 5 — EXTRACT ACTIONABLES
*Maximum 3 outputs per piece of content. More than 3 means we didn't prioritize.*

For each actionable, assign to exactly one bucket:

| Bucket | What Goes Here | Where It Goes |
|--------|---------------|---------------|
| **BUILD** | Something J5 should add or change | J5_BUILD_ROADMAP.md |
| **CREATE** | F&F content, product, or framework idea | brain/ff/content_pipeline.md |
| **FORM** | Personal practice, habit, or conviction | brain/personal/formation.md |
| **DEPLOY** | Something Curtis can use immediately | Todoist task |

### Phase 6 — CONNECT & TAG
*Nothing lives in isolation. Everything connects.*

1. Tag the synthesis against existing files:
   - Which F&F framework does this touch? (Loop / Faces / Simul)
   - Which J5 agent would use this? (Shepherd / Scribe / Catalyst / etc.)
   - Which life domain? (Family / Ministry / Business / Personal)
2. Update `brain/inspiration/INDEX.md` with one-line summary + tags
3. If significant enough: update MEMORY.md with key insight
4. Commit everything to git

## Output Standards

Every Refinery run produces:
1. `research/` folder with raw source files
2. `synthesis.md` with structured analysis
3. `curtis_synthesis.md` with Curtis's own perspective
4. 1-3 tagged actionables routed to the right bucket
5. Updated `INDEX.md` entry

Time to complete: 15-30 minutes for standard content. 1-2 hours for deep dives.

- Reference specific sections. Be precise.
- Never settle for the first version of an idea.
