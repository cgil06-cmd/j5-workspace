## ADDING A NEW API KEY (Secure Process)

### From iPhone (current standard):
1. Generate the key on the service's website
2. Copy it into Dashlane under "J5 Services" FIRST
3. Go to privnote.com → paste the key → tap "Create Note"
4. Send the Privnote link to J5 in Telegram
5. J5 reads it (self-destructs immediately), updates `~/.openclaw/.env`
6. Done — key never stored in chat history

### From Mac (preferred when at desk):
1. Generate the key on the service's website
2. Copy it into Dashlane under "J5 Services" FIRST
3. Open Terminal, run:
   ```
   nano ~/.openclaw/.env
   ```
4. Add the line: `SERVICE_NAME_API_KEY=your-key-here`
5. Save (Ctrl+X, Y, Enter)
6. Run: `chmod 600 ~/.openclaw/.env`
7. Key never touches any chat

### Via Tailscale + Termius (remote secure access):
1. Ensure Tailscale is active on iPhone and Mac
2. Open Termius → connect to Mac via Tailscale private IP
3. Run the nano command above directly on Mac
4. Key added locally, never in chat

## REVOKING A COMPROMISED API KEY
1. Go to the service immediately → API Keys → delete the exposed key
2. Generate a new one
3. Follow "Adding a New API Key" process above
4. Note the incident in `memory/YYYY-MM-DD.md`

## ADDING A NEW SERVICE/INTEGRATION
1. Check if service has API key or OAuth (preferred)
2. If password-only: create dedicated credentials in Dashlane under "J5 Services"
3. Use a unique password generated by Dashlane — nothing shared with other accounts
4. Add service to TOOLS.md with sensitivity rating
5. Flag any sensitive data J5 will now have access to

## SHARING SENSITIVE INFO WITH J5
**Never share in regular Telegram chat:**
- API keys
- Passwords
- SSNs
- Financial account numbers

**Use instead:**
- Privnote.com (one-time self-destructing link) — from iPhone
- Direct Terminal entry on Mac — preferred
- Tailscale + Termius SSH — most secure remote option
